Requisição de Transporte

MANUAL DO USUÁRIO

Introdução

Este manual instrui o usuário do SIGA-AD a fazer requisições internas de transporte de passageiros e/ou carga em veículos próprios da UFJF.



Visão Geral do Processo de Requisição

As requisições feitas pelo sistema SIGA-AD devem ser avaliadas em primeira instância pela Unidade Orçamentária (UO) à qual o usuário está diretamente ligado. Esta UO é conhecida no sistema como UO Requisitante. Depois de aprovado o pedido, o mesmo segue para o destino específico, conhecido como UO Executante (neste caso, a Gerência de Transporte). Após a execução do serviço, o usuário deve, obrigatóriamente, confirmá-lo e avaliá-lo.

Toda a operação da requisição (do pedido inicial até a confirmação e avaliação final) é feita através de uma série de telas em um navegador (browser), acessadas pelo usuário via Internet.



Telas de Requisição

Após entrar no SIGA com seu login e sua senha, surgirá a tela abaixo. No Menu Principal, escolha Administração.



O usuário requisitante terá a opção Requisições (futuramente, outras opções, como Consultas e Relatórios estarão disponíveis também a partir desta tela). Escolha Requisições.



A figura abaixo mostra a tela inicial de qualquer requisição. As possibilidades são: colocar um número de requisição para ver seus dados, procurar por uma requisição (no ícone com uma lente de aumento), inserir uma nova requisição e listar requisições existentes. O que nos interessa é criar uma nova requisição; para tal, escolha Inserir Requisição.



A primeira coisa a ser definida é o tipo de requisição (hotel, restaurante, diária, etc.). Neste caso, escolha VEICULO.



Finalmente, surge a tela específica de requisição de veículo. O primeiro item pedido é o CPF do passageiro. Importante: as requisições são individuais; caso haja mais passageiros, serão especificados ao final. Escreva o CPF do passageiro e clique no botão Procurar.



Se o passageiro não existir (ou seja: se o CPF dele não estiver cadastrado na base de dados do SIGA), serão pedidos os DADOS DO PASSAGEIRO. Serão exibidos também outros dois grupos de dados, denominados ORIGEM e DESTINO, que devem ser preenchidos pelo requisitante Após o cadastro, clique em Enviar.




Após o envio, surge a tela para confirmação da requisição. Existem três possibilidades: Confirmar, Confirmar e requerer para outro passageiro e Cancelar.




Caso se opte por requerer outro passageiro (mesma origem e destino), é pedido outro CPF (repare que o número da requisição é individual, um para cada passageiro).



Se, após digitar o CPF, o sistema verificar que o passageiro já existe (se o CPF dele já estiver cadastrado na base de dados do SIGA), surge a tela abaixo, com os dados já preenchidos (se não existir, os dados do passageiro são pedidos, como vimos para o exemplo anterior).



Após Enviar, surgirá a tela de confirmação e o processo se repete, até que todos os passageiros tenham sido especificados.



Após o último passageiro ser cadastrado e confirmado, termina o processo do pedido. Veja tela abaixo:



Depois de pedida, a requisição segue para a UO Requisitante. O Gestor da UO pode aceitá-la, recusá-la ou devolvê-la (caso haja alguma coisa errada no pedido).
Se a requisição for aceita, segue para a UO Executante, que poderá apenas aceitá-la ou recusá-la. O acompanhamento da requisição (seu andamento) pode sempre ser feito através da primeira tela de requisição (através do número da requisição ou na opção de Listar Requisições).
Após a execução de uma requisição, o usuário requisitante deve confirmar e avaliar o serviço. Este passo é obrigatório e, se houver pendência nesta avaliação final, o usuário fica impedido de fazer nova requisição.
